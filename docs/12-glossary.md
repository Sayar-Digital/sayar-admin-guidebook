# 12. Glossary

- **Onboarding requirement**: A required setup condition that must be completed for onboarding progression.
- **Structural blocker**: A non-transactional condition that blocks healthy operation (for example disconnected WhatsApp).
- **Operational incident**: A transaction/runtime issue (for example failed payments).
- **Needs Attention**: Queue for merchants currently showing meaningful risk signals.
- **Health score**: A 0-100 risk score derived from fixed penalty conditions.
- **Readiness check**: Validation checks for provider capability and setup completeness.
- **Template stale**: Template exists but does not match required current definition.
